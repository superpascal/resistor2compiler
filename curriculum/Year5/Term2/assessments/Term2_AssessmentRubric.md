# Year 5, Term 2: Assessment Rubric

## Overview

This rubric assesses student progress through Term 2 of Year 5, covering compiler implementation, Pascal program development, and OS extensions.

## Assessment Components

### 1. Compiler Implementation (Weeks 13-16) - 35%

#### Practical Work
- **Compiler Configuration** (12%)
  - Configures compiler for 65C816
  - Sets calling conventions
  - Links runtime library
  - Tests basic compilation
  - **Exceeds**: Optimized configuration

- **Compiler Extension** (12%)
  - Extends compiler with features
  - Implements intrinsics
  - Tests extensions
  - Documents changes
  - **Exceeds**: Multiple extensions

- **Compiler Verification** (11%)
  - Tests compiler thoroughly
  - Verifies code generation
  - Runs test suite
  - Documents results
  - **Exceeds**: Comprehensive testing

#### Understanding
- Explains compiler structure clearly
- Understands code generation
- Can extend compiler
- Understands runtime integration

### 2. Pascal Program Development (Weeks 17-20) - 35%

#### Practical Work
- **Basic Programs** (12%)
  - Develops basic Pascal programs
  - Tests on hardware
  - Documents programs
  - **Exceeds**: Advanced programs

- **Graphics/Sound Demos** (12%)
  - Creates graphics demos
  - Creates sound demos
  - Tests on hardware
  - Documents demos
  - **Exceeds**: Advanced demos

- **Game Development** (11%)
  - Develops simple games
  - Tests on hardware
  - Presents showcase
  - Documents games
  - **Exceeds**: Complex games

#### Understanding
- Explains Pascal programming clearly
- Understands hardware integration
- Can develop graphics/sound programs
- Can develop games

### 3. OS Extensions and Documentation (Weeks 21-24) - 30%

#### Practical Work
- **Monitor Extensions** (10%)
  - Extends monitor functionality
  - Implements new commands
  - Tests extensions
  - Documents extensions
  - **Exceeds**: Advanced features

- **File System** (10%)
  - Implements file system support
  - Tests file operations
  - Integrates with monitor
  - Documents implementation
  - **Exceeds**: Advanced file system

- **Documentation and Reflection** (10%)
  - Creates comprehensive documentation
  - Writes reflection
  - Completes system
  - Presents final system
  - **Exceeds**: Publication-quality docs

#### Understanding
- Explains OS concepts clearly
- Understands file systems
- Can create documentation
- Reflects on journey

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of software stack
- Excellent documentation
- Complete system working
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of software stack
- Good documentation
- System mostly working
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of software stack
- Basic documentation
- System partially working
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- System not working
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly compiler/program reviews
- Extension checkpoints
- Documentation reviews
- Peer reviews

### Summative Assessment
- Compiler extension project
- Pascal program showcase
- OS extensions project
- Complete system demonstration

## Documentation Requirements

### Lab Reports
- Compiler configuration
- Extension implementations
- Program source code
- Test results
- OS extensions
- Documentation
- Reflection

## Extension Opportunities

Students can extend their work by:
- Advanced compiler features
- Complex programs
- Advanced OS features
- Help other students
