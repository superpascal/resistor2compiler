# Year 4, Term 1: Assessment Rubric

## Overview

This rubric assesses student progress through Term 1 of Year 4, covering SAP-1 improvements, instruction set expansion, and memory expansion.

## Assessment Components

### 1. Review and SAP-1 Improvements (Weeks 1-3) - 35%

#### Practical Work
- **System Review** (12%)
  - Reviews SAP-1 comprehensively
  - Identifies issues correctly
  - Documents findings
  - **Exceeds**: Helps others review

- **Signal Integrity** (12%)
  - Improves signal integrity
  - Adds decoupling capacitors
  - Tests improvements
  - **Exceeds**: Advanced optimizations

- **Timing Improvements** (11%)
  - Improves timing and clock
  - Adds clock halt (if needed)
  - Tests improvements
  - **Exceeds**: Advanced timing optimization

#### Understanding
- Explains improvement concepts clearly
- Understands signal integrity
- Understands timing issues
- Can troubleshoot systematically

### 2. Expanded Instruction Set (Weeks 4-6) - 35%

#### Practical Work
- **Instruction Design** (12%)
  - Designs new instructions correctly
  - Plans microcode sequences
  - Documents designs
  - **Exceeds**: Designs multiple instructions

- **Microcode Expansion** (12%)
  - Expands microcode ROM
  - Updates address formation
  - Tests expansion
  - **Exceeds**: Optimizes expansion

- **Instruction Implementation** (11%)
  - Programs new microcode
  - Tests new instructions
  - Writes programs using them
  - **Exceeds**: Creates instruction library

#### Understanding
- Explains instruction expansion clearly
- Understands microcode design
- Can program microcode
- Understands ISA evolution

### 3. Memory Expansion (Weeks 7-9) - 30%

#### Practical Work
- **PC/MAR Extension** (10%)
  - Extends PC to 8 bits
  - Extends MAR to 8 bits
  - Tests 8-bit addressing
  - **Exceeds**: Optimizes extension

- **Memory Connection** (10%)
  - Connects larger memory
  - Implements address decoding
  - Tests memory access
  - **Exceeds**: Advanced decoding

- **Harvard Architecture** (10%)
  - Implements ROM/RAM separation
  - Tests Harvard architecture
  - Verifies complete system
  - **Exceeds**: Optimizes memory map

#### Understanding
- Explains memory expansion clearly
- Understands Harvard architecture
- Can implement address decoding
- Understands memory mapping

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of enhancements
- Excellent documentation
- Enhanced system working perfectly
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of enhancements
- Good documentation
- Enhanced system mostly working
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of enhancements
- Basic documentation
- Enhanced system partially working
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- Enhanced system not working
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly improvement testing
- Instruction design reviews
- Memory expansion verification
- Integration testing

### Summative Assessment
- Improvement project
- Instruction expansion project
- Memory expansion project
- Enhanced system demonstration

## Documentation Requirements

### Lab Reports
- Improvement logs
- Instruction designs
- Microcode tables
- Memory map designs
- Test results
- System documentation

## Extension Opportunities

Students can extend their work by:
- Advanced improvements
- More instructions
- Further memory expansion
- System optimization
