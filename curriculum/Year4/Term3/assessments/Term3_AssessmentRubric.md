# Year 4, Term 3: Assessment Rubric

## Overview

This rubric assesses student progress through Term 3 of Year 4, covering memory banking, I/O subsystems, and graphics/sound research.

## Assessment Components

### 1. Memory and Banking (Weeks 1-4) - 35%

#### Practical Work
- **Expansion Planning** (12%)
  - Plans memory expansion
  - Designs memory map
  - Documents plan
  - **Exceeds**: Optimized plan

- **Bank Implementation** (12%)
  - Implements bank switching
  - Tests bank switching
  - Verifies operation
  - **Exceeds**: Advanced features

- **Memory Testing** (11%)
  - Tests all banks
  - Verifies bank isolation
  - Documents results
  - **Exceeds**: Comprehensive testing

#### Understanding
- Explains banking concepts clearly
- Understands 24-bit addressing
- Can implement bank switching
- Understands memory mapping

### 2. I/O Subsystems (Weeks 5-8) - 35%

#### Practical Work
- **ACIA Integration** (12%)
  - Integrates ACIA
  - Tests serial communication
  - Verifies operation
  - **Exceeds**: Advanced features

- **VIA and LCD Integration** (12%)
  - Integrates VIA
  - Interfaces LCD
  - Tests I/O
  - **Exceeds**: Multiple devices

- **Complete I/O System** (11%)
  - Integrates all I/O
  - Tests complete system
  - Documents I/O map
  - **Exceeds**: Optimized system

#### Understanding
- Explains I/O concepts clearly
- Understands memory-mapped I/O
- Can integrate I/O devices
- Understands I/O programming

### 3. Graphics and Sound Research (Weeks 9-12) - 30%

#### Practical Work
- **Research** (10%)
  - Researches video options
  - Researches audio options
  - Documents research
  - **Exceeds**: Comprehensive research

- **Prototyping** (10%)
  - Prototypes selected solutions
  - Tests prototypes
  - Evaluates feasibility
  - **Exceeds**: Multiple prototypes

- **Proposal Development** (10%)
  - Creates complete proposals
  - Plans implementation
  - Estimates costs
  - **Exceeds**: Detailed proposals

#### Understanding
- Explains options clearly
- Understands trade-offs
- Can evaluate solutions
- Understands integration

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of systems
- Excellent documentation
- Complete system working
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of systems
- Good documentation
- System mostly working
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of systems
- Basic documentation
- System partially working
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- System not working
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly subsystem testing
- Integration checkpoints
- Research reviews
- Prototype evaluations

### Summative Assessment
- Memory banking project
- I/O integration project
- Graphics/sound research proposals
- Complete system demonstration

## Documentation Requirements

### Lab Reports
- Expansion plans
- Bank implementation
- Memory maps
- I/O integration
- I/O maps
- Research documentation
- Prototype results
- Proposals

## Extension Opportunities

Students can extend their work by:
- Advanced memory expansion
- Additional I/O devices
- Advanced prototypes
- Cost optimization
- Help other students
