# Year 4, Term 2: Assessment Rubric

## Overview

This rubric assesses student progress through Term 2 of Year 4, covering 65C816 introduction, system design, and prototype construction.

## Assessment Components

### 1. 65C816 Introduction (Weeks 13-16) - 35%

#### Practical Work
- **Architecture Study** (12%)
  - Studies 65C816 architecture
  - Understands key features
  - Documents findings
  - **Exceeds**: Deep research, helps others

- **SAP-1 Comparison** (12%)
  - Compares SAP-1 to 65C816
  - Maps components correctly
  - Documents comparison
  - **Exceeds**: Advanced comparison

- **Addressing Modes** (11%)
  - Understands addressing modes
  - Compares to SAP-1
  - Documents understanding
  - **Exceeds**: Advanced modes

#### Understanding
- Explains 65C816 architecture clearly
- Understands comparison to SAP-1
- Can map components
- Understands addressing modes

### 2. System Design (Weeks 17-20) - 35%

#### Practical Work
- **Architecture Design** (12%)
  - Designs system architecture
  - Plans components
  - Documents design
  - **Exceeds**: Advanced features

- **Memory Map and Decoding** (12%)
  - Designs memory map
  - Designs address decoding
  - Documents design
  - **Exceeds**: Optimized design

- **Schematic Creation** (11%)
  - Creates complete schematic
  - Uses KiCad correctly
  - Reviews and refines
  - **Exceeds**: Publication-quality

#### Understanding
- Explains system design clearly
- Understands memory mapping
- Understands address decoding
- Can create professional schematics

### 3. Breadboard Prototype (Weeks 21-24) - 30%

#### Practical Work
- **Power, Clock, Reset** (10%)
  - Builds power/clock/reset
  - Tests circuits
  - Verifies operation
  - **Exceeds**: Optimized circuits

- **Memory Connection** (10%)
  - Connects memory correctly
  - Tests memory access
  - Verifies operation
  - **Exceeds**: Advanced memory

- **CPU Verification** (10%)
  - Integrates system
  - Verifies CPU operation
  - Tests complete system
  - **Exceeds**: Advanced testing

#### Understanding
- Explains prototype construction
- Understands testing procedures
- Can debug prototype
- Understands CPU operation

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of 65C816
- Excellent documentation
- Prototype working correctly
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of 65C816
- Good documentation
- Prototype mostly working
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of 65C816
- Basic documentation
- Prototype partially working
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- Prototype not working
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly architecture study
- Design reviews
- Construction checkpoints
- Integration testing

### Summative Assessment
- Architecture comparison project
- System design project
- Prototype construction project
- Complete prototype demonstration

## Documentation Requirements

### Lab Reports
- Architecture summaries
- Comparison tables
- Component mappings
- System designs
- Schematics
- Prototype documentation
- Test results

## Extension Opportunities

Students can extend their work by:
- Advanced system features
- Optimize designs
- Add expansion capabilities
- Help other students
