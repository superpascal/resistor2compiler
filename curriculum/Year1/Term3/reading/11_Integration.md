## Program Execution

### Writing Programs

**Simple Program**: Add two numbers
1. Load A from memory[5] (value: 7)
2. Load B from memory[6] (value: 9)
3. Add A + B
4. Store result to memory[7]
5. Expected: memory[7] = 16

**Program Steps**:
- Each step = sequence of control signals
- Manual or automated execution
- Observe data flow
- Verify results

### Executing Programs

**Step-by-step**:
1. Set control signals for step 1
2. Observe data flow
3. Verify operation
4. Move to next step
5. Repeat until complete

**Verification**:
- Check intermediate results
- Verify final result
- Debug if needed
- Document execution

## Troubleshooting

### Systematic Approach

**1. Identify Problem**:
- What doesn't work?
- When does it fail?
- What's the symptom?

**2. Isolate Subsystem**:
- Test subsystems independently
- Find which subsystem has problem
- Narrow down issue

**3. Check Connections**:
- Verify all connections
- Check power
- Check signals
- Check timing

**4. Fix and Verify**:
- Fix identified problem
- Test again
- Verify solution
- Document fix

### Common Issues

**Bus Contention**:
- Multiple drivers active
- Check enable signals
- Verify only one active

**Timing Issues**:
- Signals not synchronized
- Check clock
- Verify timing

**Wrong Data**:
- Check connections
- Verify signals
- Check logic
