# Year 3, Term 1: Assessment Rubric

## Overview

This rubric assesses student progress through Term 1 of Year 3, covering data path construction for the SAP-1 CPU.

## Assessment Components

### 1. Review and ALU Subsystem (Weeks 1-3) - 35%

#### Practical Work
- **Year 1-2 Review** (10%)
  - Demonstrates understanding of foundations
  - Can recall key concepts
  - Ready for CPU construction
  - **Exceeds**: Helps others review

- **Adder Construction** (12%)
  - Builds 4-bit adder correctly
  - Tests addition operations
  - Verifies results
  - **Exceeds**: Builds from gates

- **Subtraction and Flags** (13%)
  - Implements subtraction correctly
  - Adds carry and zero flags
  - Tests all operations
  - **Exceeds**: Adds more flags

#### Understanding
- Explains ALU concepts clearly
- Understands two's complement
- Understands flag generation
- Can troubleshoot ALU issues

### 2. Register File and Buses (Weeks 4-6) - 35%

#### Practical Work
- **Register Construction** (15%)
  - Builds Register A correctly
  - Builds Register B correctly
  - Tests register operations
  - **Exceeds**: Adds more registers

- **Bus Implementation** (20%)
  - Implements tri-state bus correctly
  - Builds bus control logic
  - Tests bus protocol
  - No bus conflicts
  - **Exceeds**: Optimizes bus design

#### Understanding
- Explains register concepts clearly
- Understands bus protocol
- Understands tri-state operation
- Can troubleshoot bus issues

### 3. Memory Subsystem (Weeks 7-9) - 30%

#### Practical Work
- **MAR Construction** (10%)
  - Builds MAR correctly
  - Connects to address bus
  - Tests address loading
  - **Exceeds**: Adds address validation

- **Memory Connection** (10%)
  - Connects memory chip correctly
  - Implements address decoding
  - Tests memory connection
  - **Exceeds**: Adds multiple chips

- **Complete Data Path** (10%)
  - Integrates all components
  - Tests complete operations
  - Verifies data path
  - **Exceeds**: Optimizes data path

#### Understanding
- Explains memory concepts clearly
- Understands MAR function
- Understands read/write operations
- Can troubleshoot memory issues

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of data path
- Excellent documentation
- Can extend concepts independently
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of data path
- Good documentation
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of data path
- Basic documentation
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly lab checkpoints
- In-class observations
- Component testing
- Integration verification

### Summative Assessment
- ALU subsystem project
- Register/bus subsystem project
- Memory subsystem project
- Complete data path demonstration

## Documentation Requirements

### Lab Reports
- Circuit diagrams
- Component lists
- Test results
- Troubleshooting notes
- Data path diagrams
- Reflection on learning

## Extension Opportunities

Students can extend their work by:
- Expanding ALU operations
- Adding more registers
- Expanding memory
- Optimizing data path
- Adding features
