# Year 3, Term 2: Assessment Rubric

## Overview

This rubric assesses student progress through Term 2 of Year 3, covering control unit construction for the SAP-1 CPU.

## Assessment Components

### 1. Program Counter and Instruction Register (Weeks 10-12) - 35%

#### Practical Work
- **Program Counter Construction** (12%)
  - Builds PC correctly
  - Connects to system
  - Tests increment, load, reset
  - **Exceeds**: Adds jump capability

- **Instruction Register Construction** (12%)
  - Builds IR correctly
  - Connects to system
  - Tests instruction loading
  - **Exceeds**: Adds instruction decoding

- **Instruction Fetch** (11%)
  - Implements fetch cycle
  - Tests fetch operation
  - Verifies PC and IR work together
  - **Exceeds**: Optimizes fetch cycle

#### Understanding
- Explains PC function clearly
- Explains IR function clearly
- Understands instruction format
- Understands fetch cycle

### 2. Microcode ROM System (Weeks 17-20) - 35%

#### Practical Work
- **Microcode Format Design** (12%)
  - Designs microcode format
  - Creates address formation
  - Maps control signals
  - **Exceeds**: Optimizes format

- **Microcode ROM Construction** (12%)
  - Builds microcode ROM system
  - Programs EEPROM correctly
  - Tests ROM operation
  - **Exceeds**: Adds more instructions

- **Microcode Execution** (11%)
  - Executes instructions via microcode
  - Verifies control signals
  - Tests multiple instructions
  - **Exceeds**: Adds conditional microcode

#### Understanding
- Explains microcode concepts clearly
- Understands address formation
- Understands signal generation
- Can program microcode

### 3. Control Unit Integration (Weeks 16-18) - 30%

#### Practical Work
- **Control-Data Path Integration** (10%)
  - Integrates control unit with data path
  - Connects all signals correctly
  - Verifies integration
  - **Exceeds**: Optimizes integration

- **Fetch-Decode-Execute Cycle** (10%)
  - Implements CPU cycle
  - Tests each phase
  - Verifies cycle operation
  - **Exceeds**: Optimizes cycle

- **Complete CPU Testing** (10%)
  - Tests complete CPU
  - Executes programs correctly
  - Debugs issues
  - **Exceeds**: Adds features

#### Understanding
- Explains CPU cycle clearly
- Understands component integration
- Can debug CPU issues
- Understands CPU operation

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of control unit
- Excellent documentation
- CPU working correctly
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of control unit
- Good documentation
- CPU mostly working
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of control unit
- Basic documentation
- CPU partially working
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- CPU not working
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly lab checkpoints
- In-class observations
- Component testing
- Integration verification

### Summative Assessment
- PC/IR subsystem project
- Microcode ROM project
- Control unit integration project
- Complete CPU demonstration

## Documentation Requirements

### Lab Reports
- Circuit diagrams
- Microcode tables
- Test results
- Troubleshooting notes
- CPU architecture diagrams
- Reflection on learning

## Extension Opportunities

Students can extend their work by:
- Adding more instructions
- Optimizing microcode
- Adding debugging features
- Expanding CPU capabilities
