# Year 3, Term 3: Assessment Rubric

## Overview

This rubric assesses student progress through Term 3 of Year 3, covering SAP-1 bring-up, machine code programming, and system completion.

## Assessment Components

### 1. SAP-1 Bring-up and Testing (Weeks 19-21) - 35%

#### Practical Work
- **System Bring-up** (12%)
  - Powers up system safely
  - Tests all subsystems
  - Integrates system correctly
  - **Exceeds**: Optimizes during bring-up

- **Hardware Debugging** (12%)
  - Uses systematic debugging
  - Solves hardware issues
  - Documents debugging process
  - **Exceeds**: Creates debugging guide

- **System Verification** (11%)
  - Verifies system operation
  - Tests comprehensively
  - Documents test results
  - **Exceeds**: Advanced testing

#### Understanding
- Explains bring-up procedures clearly
- Understands debugging methodology
- Can troubleshoot systematically
- Understands system operation

### 2. Machine Code Programming (Weeks 22-24) - 35%

#### Practical Work
- **Instruction Set Understanding** (10%)
  - Understands all instructions
  - Can encode instructions
  - Understands program structure
  - **Exceeds**: Designs new instructions

- **Program Development** (12%)
  - Develops programs correctly
  - Converts to machine code
  - Loads programs into memory
  - **Exceeds**: Creates program library

- **Program Execution** (13%)
  - Executes programs correctly
  - Debugs programs systematically
  - Runs multiple programs
  - **Exceeds**: Complex programs

#### Understanding
- Explains machine code clearly
- Understands program execution
- Can debug programs
- Understands CPU operation

### 3. System Completion (Weeks 25-27) - 30%

#### Practical Work
- **Final Testing** (10%)
  - Tests system comprehensively
  - Verifies all operations
  - Documents test results
  - **Exceeds**: Advanced testing

- **Documentation** (10%)
  - Creates professional documentation
  - All documentation types complete
  - Documentation is clear and useful
  - **Exceeds**: Publication-quality

- **Presentation and Portfolio** (10%)
  - Presents system successfully
  - Demonstrates programs
  - Completes portfolio
  - **Exceeds**: Outstanding presentation

#### Understanding
- Explains complete system clearly
- Understands Year 4 requirements
- Assesses own readiness
- Reflects on learning journey

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of CPU
- Excellent documentation and presentation
- CPU working perfectly
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of CPU
- Good documentation and presentation
- CPU mostly working
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of CPU
- Basic documentation
- CPU partially working
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- CPU not working
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly testing and debugging
- Program development progress
- Documentation progress
- Presentation preparation

### Summative Assessment
- Bring-up project
- Programming project (multiple programs)
- Final documentation
- System presentation
- Complete portfolio

## Documentation Requirements

### Final Portfolio
- Complete system documentation
- All lab reports
- Program examples with machine code
- Circuit diagrams
- Photos of system
- Reflection on learning journey
- Year 3 achievements

## Year 4 Readiness

**Technical Readiness**:
- Can build complete CPU
- Can program in machine code
- Can debug systems
- Understands CPU operation
- Ready for enhanced systems

**Practical Readiness**:
- Has working SAP-1 CPU
- Can write and run programs
- Can document systems
- Can present work
- Ready for Year 4
