# Year 2, Term 1: Assessment Rubric

## Overview

This rubric assesses student progress through Term 1 of Year 2, covering memory concepts, control systems, and system integration.

## Assessment Components

### 1. Memory Concepts (Weeks 1-3) - 30%

#### Practical Work
- **MAR Construction** (10%)
  - Builds 8-bit MAR correctly
  - Displays address on LEDs
  - Can load address from switches
  - **Exceeds**: Adds address validation or error checking

- **Memory Connection** (10%)
  - Connects memory chip correctly
  - Connects MAR to memory address pins
  - Uses tri-state buffers appropriately
  - **Exceeds**: Implements address decoding

- **Memory Operations** (10%)
  - Can read data from memory
  - Can write data to memory
  - Verifies operations correctly
  - Documents read/write operations
  - **Exceeds**: Implements memory test routines

#### Understanding
- Explains memory concepts clearly
- Understands address vs data
- Understands read vs write operations
- Can troubleshoot memory issues

### 2. Control Concepts (Weeks 4-6) - 30%

#### Practical Work
- **State Machine Construction** (10%)
  - Builds state machine correctly
  - States cycle properly
  - Displays states on LEDs
  - **Exceeds**: Implements conditional transitions

- **Control Signal Generation** (10%)
  - Generates control signals correctly
  - Signals coordinated properly
  - No bus conflicts
  - **Exceeds**: Optimizes signal generation

- **Sequencer Operation** (10%)
  - Builds sequencer for operation
  - Executes operation sequence correctly
  - Documents signal timing
  - **Exceeds**: Implements multiple operation types

#### Understanding
- Explains state machines clearly
- Understands control signal coordination
- Understands sequencer operation
- Can design operation sequences

### 3. Integration and Review (Weeks 7-9) - 40%

#### Practical Work
- **System Integration** (15%)
  - Integrates all components correctly
  - All connections verified
  - System powers up without issues
  - **Exceeds**: Optimizes system layout

- **Manual Control Setup** (10%)
  - Sets up manual control switches
  - All signals accessible
  - Control reference created
  - **Exceeds**: Adds signal validation

- **Program Execution** (15%)
  - Executes program manually
  - All steps completed correctly
  - Result verified
  - Documents execution process
  - **Exceeds**: Executes multiple programs

#### Understanding
- Explains system architecture
- Understands data flow
- Understands signal coordination
- Can troubleshoot system issues
- Reflects on learning journey

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of all concepts
- Excellent documentation and reflection
- Can extend concepts independently
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of concepts
- Good documentation
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of concepts
- Basic documentation
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly lab checkpoints
- In-class observations
- Quick quizzes
- Peer discussions

### Summative Assessment
- Lab reports (weeks 1-3, 4-6)
- System integration project (weeks 7-9)
- Final demonstration
- Written reflection

## Documentation Requirements

### Lab Reports
- Circuit diagrams
- Observations and results
- Troubleshooting notes
- Reflection on learning

### Final Project
- System architecture diagram
- Component list and connections
- Control signal reference
- Program execution procedures
- Troubleshooting guide
- Reflection on learning journey

## Extension Opportunities

Students can extend their work by:
- Implementing address decoding for multiple chips
- Building more complex state machines
- Creating memory test routines
- Designing new instructions
- Optimizing system layout
- Creating advanced documentation
