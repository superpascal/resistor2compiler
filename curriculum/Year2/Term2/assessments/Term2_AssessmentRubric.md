# Year 2, Term 2: Assessment Rubric

## Overview

This rubric assesses student progress through Term 2 of Year 2, covering advanced state machines, address decoding, and I/O systems.

## Assessment Components

### 1. Advanced State Machines and Microcode (Weeks 10-12) - 35%

#### Practical Work
- **Complex State Machine** (12%)
  - Builds state machine for multiple operations
  - States cycle correctly
  - Generates control signals properly
  - **Exceeds**: Implements conditional transitions

- **Microcode ROM** (12%)
  - Sets up EEPROM as microcode ROM
  - Programs microcode correctly
  - Address formation works
  - **Exceeds**: Optimizes microcode sequences

- **Instruction Decoding** (11%)
  - Builds Instruction Register
  - Implements instruction decoding
  - Executes instructions via microcode
  - **Exceeds**: Adds multiple instructions

#### Understanding
- Explains microcode concepts clearly
- Understands instruction decoding
- Can design microcode sequences
- Can troubleshoot microcode systems

### 2. Address Decoding and Memory Mapping (Weeks 13-15) - 35%

#### Practical Work
- **Address Decoding** (12%)
  - Implements address decoding logic
  - Generates chip select signals correctly
  - No address conflicts
  - **Exceeds**: Optimizes decoding logic

- **Multiple Memory Chips** (12%)
  - Connects multiple memory chips
  - Address decoding works correctly
  - Can access all chips
  - **Exceeds**: Expands to more chips

- **Memory-Mapped I/O** (11%)
  - Designs I/O address space
  - Implements I/O interfaces
  - I/O read/write works correctly
  - **Exceeds**: Adds multiple I/O devices

#### Understanding
- Explains address decoding clearly
- Understands memory map design
- Can design memory maps
- Understands memory-mapped I/O

### 3. Input/Output Systems (Weeks 16-18) - 30%

#### Practical Work
- **Input Interface** (10%)
  - Builds input interface
  - Can read input devices
  - Interface works correctly
  - **Exceeds**: Adds multiple input types

- **Output Interface** (10%)
  - Builds output interface
  - Can control output devices
  - Interface works correctly
  - **Exceeds**: Adds display interfaces

- **Data Direction Control** (10%)
  - Implements bidirectional I/O
  - Data direction control works
  - Can change direction
  - **Exceeds**: Bit-level direction control

#### Understanding
- Explains I/O concepts clearly
- Understands data direction
- Can interface with devices
- Understands I/O timing

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of all concepts
- Excellent documentation
- Can extend concepts independently
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of concepts
- Good documentation
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of concepts
- Basic documentation
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly lab checkpoints
- In-class observations
- Quick quizzes
- Peer discussions

### Summative Assessment
- Lab reports (weeks 10-12, 13-15, 16-18)
- Microcode programming project
- Memory map design project
- I/O system project

## Documentation Requirements

### Lab Reports
- Circuit diagrams
- Microcode listings
- Memory maps
- I/O interface designs
- Test results
- Troubleshooting notes
- Reflection on learning

## Extension Opportunities

Students can extend their work by:
- Implementing more complex state machines
- Adding more instructions to microcode
- Designing larger memory maps
- Adding more I/O devices
- Optimizing systems
