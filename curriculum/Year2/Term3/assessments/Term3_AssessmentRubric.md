# Year 2, Term 3: Assessment Rubric

## Overview

This rubric assesses student progress through Term 3 of Year 2, covering complex program execution, system optimization, and final integration.

## Assessment Components

### 1. Complex Program Execution (Weeks 25-28) - 35%

#### Practical Work
- **Program Design** (12%)
  - Designs complex multi-step program
  - Plans data flow correctly
  - Sequences operations properly
  - **Exceeds**: Optimizes program design

- **Program Execution** (12%)
  - Executes program manually correctly
  - Coordinates all control signals
  - Verifies each step
  - **Exceeds**: Executes multiple programs

- **Testing and Documentation** (11%)
  - Tests program thoroughly
  - Documents execution process
  - Creates program documentation
  - **Exceeds**: Creates program library

#### Understanding
- Explains program design clearly
- Understands execution flow
- Can debug program issues
- Can optimize programs

### 2. System Optimization and Debugging (Weeks 29-32) - 35%

#### Practical Work
- **Signal Integrity** (12%)
  - Improves signal integrity
  - Adds decoupling capacitors
  - Tests improvements
  - **Exceeds**: Advanced signal optimization

- **Timing Optimization** (12%)
  - Optimizes system timing
  - Improves clock distribution
  - Tests timing improvements
  - **Exceeds**: Advanced timing optimization

- **Systematic Debugging** (11%)
  - Uses systematic debugging method
  - Solves complex issues
  - Documents debugging process
  - **Exceeds**: Creates debugging guide

#### Understanding
- Explains optimization concepts
- Understands debugging methodology
- Can troubleshoot systematically
- Can validate improvements

### 3. Final Integration and Preparation (Weeks 33-36) - 30%

#### Practical Work
- **System Review** (10%)
  - Completes comprehensive review
  - Documents system architecture
  - Creates system diagrams
  - **Exceeds**: Advanced documentation

- **Documentation** (10%)
  - Creates professional documentation
  - All documentation types complete
  - Documentation is clear and useful
  - **Exceeds**: Publication-quality documentation

- **Presentation and Portfolio** (10%)
  - Presents system successfully
  - Demonstrates system operation
  - Completes portfolio
  - **Exceeds**: Outstanding presentation

#### Understanding
- Explains complete system clearly
- Understands Year 3 requirements
- Assesses own readiness
- Reflects on learning journey

## Overall Assessment

### Grade Descriptors

**Excellent (A)**:
- All practical work completed successfully
- Deep understanding of all concepts
- Excellent documentation and presentation
- Ready for Year 3 CPU construction
- Helps other students

**Good (B)**:
- Most practical work completed successfully
- Good understanding of concepts
- Good documentation and presentation
- Mostly ready for Year 3
- Can apply concepts with guidance

**Satisfactory (C)**:
- Basic practical work completed
- Basic understanding of concepts
- Basic documentation
- Needs some preparation for Year 3
- Needs guidance to apply concepts

**Needs Improvement (D)**:
- Some practical work incomplete
- Limited understanding
- Incomplete documentation
- Needs significant preparation for Year 3
- Significant guidance needed

## Assessment Methods

### Formative Assessment
- Weekly lab checkpoints
- In-class observations
- Progress reviews
- Peer discussions

### Summative Assessment
- Complex program project
- Optimization project
- Final documentation
- System presentation
- Complete portfolio

## Documentation Requirements

### Final Portfolio
- Complete system documentation
- All lab reports
- Project photos
- Circuit diagrams
- Program examples
- Reflection on learning journey
- Year 3 readiness assessment

## Year 3 Readiness

**Technical Readiness**:
- Can build all components
- Can integrate systems
- Can debug issues
- Understands all concepts
- Ready for CPU construction

**Practical Readiness**:
- Has working complete system
- Can execute programs
- Can document systems
- Can present work
- Ready for next challenge
